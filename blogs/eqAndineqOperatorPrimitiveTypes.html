What is the deal with == and === in primitive types?

When working in C/C++ on can only compare ascii value of char with integer, but can not compare implied value of the char with the integer.

Example:
"1" is equal to 49 in decimal
"1" is not equal to 1 in decimal

In JavaScript, we can compare the implied value in one type(in our case string) with another type(in our case integer).

Coming to == and ===

We use "==" when we want JavaScript to make type of both place holder values same before comparision.
Therefore, "1" == 1 will return true.

With "===" no conversion of type is done and place holder values compared as it is.
Therefore, "1" === 1 will return false.

What about != and !==

Similar logic is applied to inequality operator as well.
"1" != 1 will return false.
"1" !== 1 will return true.


Above explaination is for primitive types, things change a bit with objects. Stay tuned, will be explaining the same in future blogs.
